# AB-900 Microsoft Learn Course Syllabus

Exam **AB-900: Microsoft 365 Copilot and Agent Administration Fundamentals** certifies you as **Microsoft 365 Certified: Copilot and Agent Administration Fundamentals**.

---

## Exam Domains and Weights

| # | Domain | Weight |
|---|--------|--------|
| 1 | Identify the core features and objects of Microsoft 365 services | 30-35% |
| 2 | Understand data protection and governance tasks for Microsoft 365 and Copilot | 35-40% |
| 3 | Perform basic administrative tasks for Copilot and agents | 25-30% |

---

## Domain 1: Core Features and Objects of Microsoft 365 Services (30-35%)

### Core M365 Objects

| Topic | Key Areas |
|-------|-----------|
| Licensing | License types (E3, E5, Business), group-based licensing, Copilot add-on |
| M365 admin center | Domain names, org settings, users, groups |
| Exchange Online | Mailboxes, distribution lists, mail flow |
| SharePoint | Sites, libraries, folders, permissions (Owner/Member/Visitor) |
| Teams | Teams, channels (standard/private/shared), policies, apps |

### Security Principles

| Topic | Key Areas |
|-------|-----------|
| Zero Trust | Verify explicitly, least privilege, assume breach |
| Authentication | MFA, passwordless, Entra ID authentication methods |
| Authorisation | RBAC, Conditional Access, SSO |
| Threat protection | Defender XDR (Endpoint, Office 365, Identity, Cloud Apps) |

### Security Features

| Topic | Key Areas |
|-------|-----------|
| Microsoft Entra ID | Users, groups, Conditional Access, SSO, PIM, app registrations |
| Identity Secure Score | Security posture score, improvement actions |
| Troubleshooting | Sign-in logs, audit logs, "What If" tool, risky sign-ins |

---

## Domain 2: Data Protection and Governance (35-40%)

### Microsoft Purview

| Topic | Key Areas |
|-------|-----------|
| Information Protection | Sensitivity labels (encryption, markings, access control) |
| DLP | Policies to detect and prevent sensitive data sharing |
| Insider Risk Management | Detect risky user activities |
| Communication Compliance | Monitor messages for policy violations |
| DSPM for AI | Discover and manage AI-related data security risks |
| Data Lifecycle Management | Retention and deletion policies |
| Data classification | SITs, trainable classifiers, content/activity explorer |
| Compliance Manager | Compliance score, assessments, improvement actions |

### Copilot Data Security

| Topic | Key Areas |
|-------|-----------|
| Data access model | Copilot uses Microsoft Graph, respects user permissions |
| Microsoft Purview integration | Sensitivity labels, DLP, Defender apply to Copilot |
| Responsible AI | Fairness, reliability, privacy, inclusiveness, transparency, accountability |
| Oversharing risks | SharePoint permissions, data access governance reports, Advanced Management |

---

## Domain 3: Copilot and Agent Administration (25-30%)

### Copilot Features

| Topic | Key Areas |
|-------|-----------|
| Built-in features | Copilot in Word, Excel, PowerPoint, Outlook, Teams, Business Chat |
| Researcher | Web and internal data search for research summaries |
| Analyst | Data analysis, insights, trends in Excel and other sources |
| Licensing | Monthly per-user vs pay-as-you-go, qualifying base licenses |

### Copilot Administration

| Topic | Key Areas |
|-------|-----------|
| License assignment | M365 admin center > Billing > Licenses |
| Feature management | Enable/disable web search, per-app toggles, plugins |
| Usage monitoring | Copilot Analytics, M365 usage reports, Viva Insights |
| Prompt management | Save, share, schedule, delete prompts |

### Agent Administration

| Topic | Key Areas |
|-------|-----------|
| Agent types | Built-in, custom (Copilot Studio), SharePoint agents, declarative |
| User access | Copilot Studio access policies, app policies |
| Agent creation | Copilot Studio, SharePoint, declarative manifests |
| Approval process | Admin approval for publishing in Power Platform admin center |
| Monitoring | M365 admin center (usage), Power Platform admin center (operations) |
