# AB-900 Glossary of Key Terms

## A

**Agent** — A task-specific AI assistant built using Copilot Studio, SharePoint, or declarative manifests. Agents can be scoped to specific data sources and actions, unlike general-purpose Copilot.

**Analyst** — A Copilot feature that works with data in Excel and other sources to provide insights, identify trends, and perform analysis.

**App permission policies** — Teams admin center policies that control which apps users can install and use within Microsoft Teams.

**App registrations** — Microsoft Entra ID feature for registering applications to use Entra ID for authentication (OAuth 2.0 / OpenID Connect).

**Audit logs** — Records of admin and user activities in Microsoft 365, including directory changes, policy modifications, and configuration updates.

## B

**Business Chat** — Copilot's cross-app conversational interface (microsoft365.com/chat and Teams) that accesses data from all M365 apps via Microsoft Graph.

## C

**Compliance Manager** — Microsoft Purview tool providing a compliance score against regulatory standards (GDPR, ISO 27001, NIST) with specific improvement actions.

**Conditional Access** — Microsoft Entra ID feature that enforces access decisions based on signals (user, location, device, risk). Uses if/then logic: if [condition], then [grant/block].

**Communication Compliance** — Microsoft Purview service that monitors messages (Teams, Exchange, Viva Engage) for policy violations such as harassment, threats, and regulatory non-compliance.

**Content explorer** — Microsoft Purview tool for browsing actual content that has been classified with sensitivity labels or sensitive information types.

**Copilot Analytics** — Dashboard showing Copilot usage metrics, adoption trends, active users, and feature-level usage statistics.

**Copilot Pages** — A collaborative, persistent canvas for editing and sharing AI-generated content within Microsoft 365.

**Copilot Studio** — Full-featured platform for building custom agents with topics, actions, knowledge sources, and authentication.

## D

**Data access governance report** — SharePoint admin center report showing sharing activity, permissions, and external access patterns across sites.

**Data classification** — The process of identifying and categorising data based on sensitivity using sensitive information types (SITs), trainable classifiers, and exact data match (EDM).

**Data Loss Prevention (DLP)** — Microsoft Purview service that detects and prevents accidental sharing of sensitive information via policies that scan content in emails, documents, and chats.

**Data Lifecycle Management** — Microsoft Purview service for managing retention and deletion policies to meet regulatory and business requirements.

**Data Security Posture Management (DSPM) for AI** — Microsoft Purview feature for discovering, monitoring, and managing AI-related data security risks across M365 and Copilot.

**Defender XDR** — Microsoft's unified extended detection and response platform across identities, endpoints, email, and cloud apps.

**Distribution list** — An Exchange Online group object that sends email to multiple recipients simultaneously.

## E

**eDiscovery** — Microsoft Purview feature for legal hold, search, and export of emails and files for litigation and compliance investigations.

**Enterprise apps** — Third-party and custom applications connected to Microsoft Entra ID for SSO and access management.

**Exchange Online** — Microsoft 365's cloud email service, managing mailboxes, distribution lists, mail flow rules, and connectors.

## G

**Group-based licensing** — Assigning licenses to an Entra ID group so all members automatically receive the assigned licenses.

## I

**Identity Secure Score** — Microsoft Entra ID metric (percentage-based) measuring identity security posture with recommendations for improvement.

**Information Protection** — Microsoft Purview service for classifying and protecting data using sensitivity labels (encryption, visual markings, access restrictions).

**Insider Risk Management** — Microsoft Purview service detecting risky user activities like data theft, security policy violations, and suspicious behavior patterns.

## M

**MFA (Multi-Factor Authentication)** — Requiring two or more verification methods for authentication: something you know (password), something you have (phone), something you are (biometric).

**Microsoft Graph** — The unified API connecting all Microsoft 365 data (mail, files, calendar, people, Teams). Copilot uses Graph to discover and retrieve relevant content.

**Microsoft Purview** — Microsoft's unified data governance, compliance, and risk management platform for Microsoft 365.

## O

**Oversharing** — When users have access to more data than they need in SharePoint. Critical for Copilot because Copilot surfaces everything the user can access.

## P

**Pay-as-you-go** — Consumption-based billing for Copilot in SharePoint and agent usage, linked to an Azure subscription instead of per-user licenses.

**PIM (Privileged Identity Management)** — Microsoft Entra ID feature providing just-in-time privileged access with activation, approval, and time-limited role elevation.

**Prompt** — An instruction or question provided to Copilot. Prompts can be saved, shared, scheduled, and deleted by users.

## R

**Researcher** — Copilot feature that searches the web and internal data to find information and generate research summaries.

**Restricted site access** — SharePoint Advanced Management feature limiting site access to members of a specific security group, preventing broader access.

**Retention policy** — Microsoft Purview policy controlling how long content is kept and when it is deleted, applied to locations like Exchange, SharePoint, and Teams.

## S

**Sensitivity label** — Microsoft Purview label that classifies and protects data with encryption, visual markings (headers, footers, watermarks), and access restrictions.

**Sensitive information type (SIT)** — Pattern-based detection rules in Purview for identifying data like credit card numbers, SSNs, and passport numbers.

**SharePoint Advanced Management** — Premium SharePoint features including restricted site access, data access governance reports, and site lifecycle management.

**Sign-in logs** — Microsoft Entra ID logs showing every authentication attempt, applied Conditional Access policies, and MFA prompts.

**SSO (Single Sign-On)** — Authentication feature allowing one set of credentials to access all connected applications without re-entering passwords.

## T

**Trainable classifiers** — AI-trained content classifiers in Purview for detecting content types like resumes, source code, contracts, and financial statements.

## W

**What If tool** — Conditional Access simulation tool that shows which policies would apply to a given sign-in scenario, helping troubleshoot access issues.

## Z

**Zero Trust** — Security model with three principles: verify explicitly, use least privilege access, and assume breach. No user or device is trusted by default.
